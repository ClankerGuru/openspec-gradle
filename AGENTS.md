# AGENTS.md — For AI Coding Assistants

## Important: Use Gradle, Not Scripts

This is a Gradle project. **Always use Gradle tasks** to perform actions — never write bash scripts, shell commands, or manual file manipulation to do something that a Gradle task already handles.

To discover what's available:

```bash
gradle tasks --group=openspec
```

Each task includes a full description of what it does, what arguments it takes, and when to use it. Read those descriptions — they are your documentation.

### Why Gradle over scripts?

- Gradle tasks are **declarative, cacheable, and reproducible**
- Tasks define their own **inputs, outputs, and dependencies**
- Running `gradle <task>` ensures the correct build environment, classpath, and configuration
- Scripts break when paths change; Gradle tasks don't

### Examples

| Instead of... | Use... |
|---|---|
| Manually creating spec files | `gradle openspecPropose --name=my-feature` |
| Running a script to generate prompts | `gradle openspecSync` |
| Deleting generated files by hand | `gradle openspecClean` |
| Moving completed specs to archive | `gradle openspecArchive --name=my-feature` |

## How the Plugin Works

- The plugin generates Markdown files (skills + commands) that AI coding assistants discover at their conventional paths (e.g. `.github/prompts/`, `.claude/skills/`)
- Generated files are ephemeral — they are added to `.gitignore` and regenerated by `gradle openspecSync`
- The source of truth for prompts is the plugin itself, not the generated files
- Updating the plugin version and re-running sync updates all prompts

## Project Structure

- `src/main/kotlin/dev/openspec/gradle/` — Plugin source code
  - `OpenSpecPlugin.kt` — Main plugin class
  - `OpenSpecExtension.kt` — DSL configuration (`openspec { }` block)
  - `tasks/` — Gradle task implementations
  - `generators/` — Skill and command file generators
  - `templates/` — Prompt template content
- `src/test/kotlin/` — Tests (unit + integration via Gradle TestKit)
- `build.gradle.kts` — Plugin build file (Kotlin DSL)

## Building & Testing

```bash
./gradlew build                    # compile + test
./gradlew publishToMavenLocal      # publish to ~/.m2 for local use
./gradlew openspecInstallGlobal    # install globally via init script
```
