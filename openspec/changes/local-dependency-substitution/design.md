# Design: Local Dependency Substitution

## Registry File: `~/.gradle/repositories.md`

Markdown table — human-readable, easy to edit, parseable:

```markdown
# Repository Registry

| Key | URL | Local Path | GAV | Substitute |
|-----|-----|------------|-----|------------|
| openspec-gradle | https://github.com/ClankerGuru/openspec-gradle.git | /data/workspace/openspec-gradle | zone.clanker:openspec-gradle | true |
| gradle-distribution | https://github.com/ClankerGuru/gradle-distribution.git | /data/workspace/gradle-distribution | zone.clanker:gradle-distribution | true |
| my-android-app | https://github.com/ClankerGuru/my-android-app.git | | | false |
```

**Fields:**
- **Key**: Short unique identifier (kebab-case)
- **URL**: Git remote URL (for reference/documentation)
- **Local Path**: Absolute path to local clone. Empty = not cloned on this machine
- **GAV**: `group:artifact` coordinates the repo publishes (comma-separated if multiple). Used for documentation/validation — Gradle discovers these automatically from `includeBuild`
- **Substitute**: `true`/`false` — whether to include this repo in composite builds

**Why Markdown?**
- Human-readable without tooling
- Git-friendly (diffable)
- Parseable (pipe-delimited table)
- You can add notes/sections around the table

## Init Script: `~/.gradle/init.d/repositories.init.gradle.kts`

Generated/updated by the plugin. Reads the registry at settings evaluation time:

```kotlin
// Auto-generated by openspec-gradle plugin
// Do not edit — managed by ./gradlew openspecInstallRepos

beforeSettings {
    val registryFile = File(System.getProperty("user.home"), ".gradle/repositories.md")
    if (!registryFile.exists()) return@beforeSettings

    // Global enable/disable
    val globalEnabled = providers.gradleProperty("zone.clanker.substitute")
        .getOrElse("false") == "true"
    if (!globalEnabled) return@beforeSettings

    // Parse Markdown table
    val repos = parseRepositoryTable(registryFile)
    
    for (repo in repos) {
        if (!repo.substitute) continue
        if (repo.localPath.isBlank()) continue
        
        val localDir = File(repo.localPath)
        if (!localDir.exists()) {
            logger.warn("OpenSpec: Registry repo '${repo.key}' local path does not exist: ${repo.localPath}")
            continue
        }
        if (!File(localDir, "settings.gradle.kts").exists() && !File(localDir, "settings.gradle").exists()) {
            logger.warn("OpenSpec: Registry repo '${repo.key}' is not a Gradle project: ${repo.localPath}")
            continue
        }
        
        includeBuild(repo.localPath)
        logger.lifecycle("OpenSpec: Substituting '${repo.key}' from local: ${repo.localPath}")
    }
}
```

## Property Hierarchy

```
zone.clanker.substitute=true   # in ~/.gradle/gradle.properties → global enable
zone.clanker.substitute=false  # in project/gradle.properties → project-level disable
```

The init script checks `providers.gradleProperty()` which respects the hierarchy.

Additionally, individual repos can be toggled:
```
zone.clanker.substitute.openspec-gradle=false  # disable specific repo even if global is on
```

## Per-project Opt-out

In the init script's `beforeSettings` block, we can't easily read the *consuming* project's `gradle.properties` before settings evaluate. Two options:

### Option A: Check at `settingsEvaluated`
Too late — `includeBuild` must be called during settings evaluation.

### Option B: Read the file directly
Same approach as the agents property — read `gradle.properties` from the project directory:

```kotlin
val projectProps = File(settings.rootDir, "gradle.properties")
val projectDisabled = readProperty(projectProps, "zone.clanker.substitute") == "false"
if (projectDisabled) return@beforeSettings
```

**Recommendation**: Option B — direct file read before calling `includeBuild`.

## Components

### 1. RepositoryRegistryParser

Parses the Markdown table into structured data:

```kotlin
data class RepositoryEntry(
    val key: String,
    val url: String,
    val localPath: String,
    val gav: List<String>,   // ["zone.clanker:openspec-gradle"]
    val substitute: Boolean
)

object RepositoryRegistryParser {
    fun parse(file: File): List<RepositoryEntry>
    fun render(entries: List<RepositoryEntry>): String  // back to Markdown
}
```

### 3. Init Script Generator

Part of `OpenSpecInstallGlobalTask` — generates `repositories.init.gradle.kts`:

The init script is **self-contained** (no plugin dependency) — it's pure Kotlin script that:
1. Reads the Markdown table
2. Parses it with inline code (no external dependency needed)
3. Calls `includeBuild` for each enabled local repo

This is important because the substitution init script runs *before* the plugin is loaded.

## File Layout

```
~/.gradle/
├── init.d/
│   ├── openspec.init.gradle.kts          # existing — applies the plugin
│   └── repositories.init.gradle.kts      # NEW — dependency substitution
├── gradle.properties                      # zone.clanker.substitute=true
└── repositories.md                        # the registry
```

## Why Two Init Scripts?

The substitution script must be self-contained (runs before any plugin classpath is available). It's pure settings-level Kotlin with no dependencies. The plugin init script needs the plugin on the classpath. Keeping them separate avoids coupling.

## Standard Clone Directory

By convention, repos are cloned to `~/workspace/<key>` or a configurable base:

```properties
zone.clanker.workspace=/data/workspace  # in ~/.gradle/gradle.properties
```

If `localPath` is empty in the registry but the directory `${workspace}/${key}` exists, the init script can auto-discover it.
